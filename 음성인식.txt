1. 안드로이드 주요기능
애플리케이션 프레임워크를 통해 제공되는 api 사용함으로써 코드를 재사용하여 효율적이고 빠르게
애플리케이션 개발 가느 
모바일용 DB인 SQLite 지원
각종 오디오, 비디오, 이미지 형식 지원
모바일 기기에 내장된 각종 하드웨어 지원

안드로이드 특징
안드로이드 핵심 커널은 리눅스로 구성
안드로이드 sdk에서 많은 라이브러리를 포함하고 있어 개발에 용이하다
오픈 소스 지햠으로 운영체제부터 관련 문서 등 거의 모든 것이 무료이다
지속적인 업그레이드 제공

안드로이드 위젯
뷰의 개요
View 클래스
화면 내에서 실제로 사용되는 것들은 모두 View 클래스의 상속을 받는다
다른 말로는 위젯이라고도 함
위젯을 담을 수 있는 위젯은 레이아웃이라고 불림
레이아웃은 ViewGroup클래스 아래에 존재

레이아웃 종류
Linear 왼쪽위부터 아래쪽 혹은 오른쪽으로 차례대로 배치
Relative : 자신이 속한 레이아웃의 상하좌우의 위치를 지정하여 배치
Table 위젯의 행과 열릐 개수를 지정한 테이블 형태로 배열
Grid 테이블과 비슷하나 행 또는 열을 확장하여 다양하게 배치할 때 더 편함
Frame 위젯들을 왼쪽 상단에 일률적으로 겹쳐서 배치하여 중복해서 보이는 효과를 지님

raw 폴더 파일 읽기 전용만 사용 가능

Activity and Intent
4대 컴포넌트
액티비티 : 화면을 구성하는 가장 기본적인 컴포넌트
서비스 : 액티비티와 상관없이 백그라운드에서 동작하는 컴포넌트
브로드캐스트 리시버 : 문자 도착, 배터리 방전 등 전체 응용프로그램이 들을 수 있도록 방송 신호를 보냄
컨텐트 프로바이더 : 응용 프로그램 사이에 데이터를 상호 공유하기 위한 컴포넌트

액티비티
일반적으로 액티비티 하나 당 xml 파일 하나를 만들어서 사용 매니패스트에 꼭 등록해야 됌

인텐트
안드로이드 4대 컴포넌트가 상호 간 데이터를 주고 받기 위한 메시지 객체
명시적, 암시적 인텐트로 구분
명시적 인텐트 : 다른 액티비티의 이름을 명활히 지정할 때 사용
암시적 인텐트 : 약속된 액션을 지정하여 안드로이드에서 제공하는 기존 응용 프로그램을 싱행하는 것

어댑터 뷰
List, ExpandableList, Grid, Spinner, Gallery등을 묶어서 표현
리스트 : 데이터를 리스트 모양으로 보여주며 리스트 중 하나를 선택하는 용도
그리드 : 사진이나 그림을 격자 모양으로 배치
갤러리 : 사진이나 이미지를 배치하고 좌우로 스크롤해서 볼 수 있음
스피너 : 드롭다운 받스와 비슷함

데이터 저장과 관리
데이터베이스 : 대용량의 데이터 집합을 체계적으로 구성해놓은 것
DBMS(데이터베이스 관리 시스템)
데이터베이스는 여러 사용자나 시스템이 서로 공유 가능해야함
데이터베이스를 관리해주는 시스템 또는 소프트웨어를 말함
크게 계층형, 망형 관계형, 객체지향형, 등의 유형으로 나뉨

SQLite: 관계형 DBMS
데이터 : 하나하나의 단편적 정보를 뜻함
DB : 테이블이 저장되는 장소, 각 데이터베이스는 서로 다른 고유한 이름이 있어야함

멀티미디어, 스레드
MediaPlayer : 음악 및 동영상 재생 클래스
스레드(Thread)
동시에 여러 작업을 수행하기 위해 사용되는 개념
스레드는 하나의 작업이 끝나기 전에 다른 작업을 동시에 진행할 수 있음
UI Thread
화면의 위젯을 변경할 때 사용
Content Provider
안드로이드는 보안상 앱에서 사용하는 데이터를 외부에서 접근이 불가능
그렇기 때문에 외부 앱에서 사용하려면 Content Provider를 만들어서 외부로 제공
URL Content Provider에서 제공하는 데이터에 접근하기 위한 주소

STT / TTS
음성에서 텍스트 / 텍스트에서 음성
음성 검색 : 음성으로 검색하는 기능
안드로이드 음성 처리 액션 : 음성 처리 액션을 통해 사용자는 음성으로 단말에 명령을 내릴 수 있음
가상 개인비서 : 알람 설정, 캘린더 업데이트, 일상 대화와 같은 다양한 테스트 수행

STT기술
신호처리 : 마이크에 입력되는 음성 언어의 캡쳐과정과 이를 컴퓨터가 처리할 수 있는 디지털 데이터로
변환 하는 아날로그 - 디지털 변환 과정 수반
음성인식 : 음성신호는 컴퓨터가 이닛ㄱ할 수 있는 언어의 모든 음소와 일치하는 세분화된 세그먼트로 분리

STT 사용방법
RecognizerIntent Class, SpeechRecognizer Class 두가지 존재
Re~ 사용이 쉬움 SpeechRecognizer : GUI 제어권을 개발자에게 더 많이 제공, 상대적 고급 기수 

TTS
- TTS는 텍스트와 음성 간 합성 기술로 글로 쓴 텍스트를 음성으로 변환하는 기술
- TTS는 시각 장애인과 언어 장애인을 위해 단말 화면의 내용을 읽어주는 용도로 널리 사용됨
- TTS 시스템의 품질은 사용자가 음성을 입력하는 방법에 따라 상당한 관련성이 있음

TTS 기술
텍스트 분석 : 합성될 텍스트를 분석하고 음성 출력을 위한 준비 단계
웨이브 포맷의 데이터 생성 : 분석된 텍스트를 음성으로 변환하느 ㄴ단계

구글 클라우드 플랫폼
GCP의 대표적 서비스 참고

타사ㅢ 퍼블릭 클라우드 서비스 : 아마존 AWS, MS Azure
클라우드 서비스
서버나 스토리지 같은 자원이 필요할떄 구글이나 아마존, MS와 같은 벤더로부터 빌려서 사용하고 비용을 지불하는 서비스

클라우드 제공 서비스 분류

- 미들웨어, 프레임 워크

애플리케이션 프로그램

구글 클라우드 플랫폼 - 구글 어시스턴트
시리와 비슷한 비서용 프로그램
자연어 분석,  머신 러닝, 음성 매칭 등 최신 기술이 있으며 안드로이드가 탑재된 거의 모든 
플랫폼에서 동작할 수 있음을 강조

특징
음성 매치
- 음성을 분석하여 사용자를 구분함
- 로그인할 필요 없이 사용자의 음성만으로 개인별 특정 서비스 활용 가능
대화형 인터페이스
- 사용자의 입력을 받아 결과를 내는 수동적인 대화형 인터페이스
다양한 플랫폼지원
SDK지원 : 개인 비서 개발 가능

Actions SDK
서버가 사용되지 않는 봇을 만드는 SDK
속도가 빠르지만 웹에서 정보를 가져올 수 없으며 앱 내 데이터만을 사용하여 응답함
DialogFlow SDK
대화형 사용자 인터페이스를 설계하고 모바일 앱, 웹 어플리케이션, 기기, 봇, 대화형 음성 응답 시스템
및 관련 용도에 통합하는데 사용되는 자연어 이해 플랫폼
DialogFlow 작동방식 : 사용자 입력 (음성->텍스트 변환) 
자연어 처리를 위한 JSON 요청으로 시작 -> JSON 형태로 전달 받아 사용자에게 응답 결과 제시

Agents
최종 사용자와 대화를 처리하는 가상 에이전트

Intents
인텐트는 대화에서 사용자의 의도를 분류하는 기능을 가짐
Training phrases : 사용자가 입력하거나 말할 수 있는 예시 문구
Parameters : 사용자의 입력의 특정 부분에서 값이 추출하기 위해 매개변수를 사용하도록 학습문구를 정의

Intent
Default Fallback Intent
사용자의 요청을 인공지능이 이해하지 못하였을 때 응답할 말을 정의
-> 이해하지 못하였을때 다시 말해달라는 문장을 정의함
Default Welcome Intent
처음 실행했을 때의 행동의 정의(기본적인 인사 문장들)
Follow-up Intent
맥락 속에서 연속적인 질의 응답을 수행하는 연계형 챗봇
상위 Intent와 하위 인텐트로 이전 대화에서 일치하는 경우 사용됨
예약과 같은 사용자로부터 다양한 정보를 연속적으로 받아야 할 떄 활용
